CHAPTER 01 · 14 min · PAID CHAPTER · PREVIEW

The terminal as a language.

Now that you can create a file, let's learn to move. A filesystem is a tree — and the commands that navigate it are the ones you'll use every single day of your career.

The tree

A filesystem isn't a list. It's a tree. It has a root — on Unix systems, / — and from there everything branches into directories that contain other directories and, at the tips, files.

When you open a terminal, you're somewhere in that tree. That “somewhere” is called the working directory. Every command sees the world relative to it. Navigating means choosing where you want to be.

▸
Mental rule
Paths that start with / are absolute — they begin at the root. Paths that don't start with / are relative — they begin where you are. Almost every “file not found” bug is a relative path treated as absolute.

The commands you'll use every day

You know three of them from chapter 00: pwd, ls, and cat. Three more, and you have the full toolkit for navigating any project in your career.

cd <path>
changes the working directory to <path>
cd ..
goes up one level (.. means “the parent”)
cd
returns to your home directory — where you land when the terminal opens
ls -la
lists everything — including hidden files (those starting with a dot)
find . -name <pattern>
searches recursively from here for a name matching the pattern
