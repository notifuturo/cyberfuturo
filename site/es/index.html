El stack, capa por capa

La mayoría de los cursos te enseñan Python desde la sintaxis hacia arriba — y asumen que ya sabes usar un terminal, un editor, y Git. Nosotros empezamos por la capa que ya reconoces y bajamos, una herramienta a la vez, hasta el metal.

  1. CAPA 6
    GUI — lo que ya conoces

    VS Code, un navegador, archivos. Las mismas interfaces que usas todos los días.

  2. CAPA 5
    Terminal & Bash

    El mismo editor, pero ahora con una terminal abierta. pwd, ls, cd, pipes, y el shell que vas a usar el resto de tu carrera.

  3. CAPA 4
    Git & GitHub

    Cada ejercicio completado es un commit real. Aprendes Git usándolo, no estudiándolo.

  4. CAPA 3
    SQL con SQLite

    CREATE, INSERT, SELECT, joins — con una base de datos real que vive en un archivo, directamente en tu Codespace.

  5. CAPA 2
    Python desde cero

    python3, variables, funciones, archivos, pip — con el intérprete que ya está configurado en tu entorno.

  6. CAPA 1
    HTTP & APIs reales

    curl, respuestas JSON, primer script que llama a una API pública. Ahora ya sabes cómo funciona Internet por dentro.

Cómo funciona

Pista: usa echo con un redireccionamiento > user@cyberfuturo:~/curriculum$ echo "hola mundo" > hola.txt user@cyberfuturo:~/curriculum$ ./cf check ✔ hola.txt existe ✔ contenido correcto ✔ lección 01 completada Siguiente: ./cf next user@cyberfuturo:~/curriculum$ █
  1. Abres el proyecto en GitHub Codespaces

    Un click desde el botón de arriba. GitHub te da una máquina virtual con VS Code, una terminal, Python, Git y SQLite ya configurados. Nada que instalar.

  2. Ejecutas ./cf start para tu primera lección

    Un pequeño programa (escrito en Python puro, sin dependencias) te muestra la tarea, las instrucciones y las pistas. Las lecciones están en la terminal porque la terminal es lo que vas a aprender.

  3. Haces la tarea con comandos reales

    Cada ejercicio es una pequeña misión: crear un archivo, encontrar uno oculto, escribir un script Python, ejecutar una consulta SQL. Usas las herramientas reales, no simulaciones.

  4. Ejecutas ./cf check para verificar

    El validador revisa el estado real del sistema de archivos, del repositorio Git, o de la salida del programa. Te dice qué pasó y qué falta.

  5. Cada lección completada es un commit en Git

    Aprendes Git sin darte cuenta: cada logro queda registrado en tu historial. Cuando termines el curso vas a tener un repositorio real con 30+ commits reales, listo para poner en tu CV.

Temario — versión 0.1

Un subconjunto pequeño, enseñado en profundidad. Cuanto más largo el temario, menos probable es que lo termines. Elegimos lecciones que son indispensables, no "nice to have".

  • 00
    Bienvenida & orientación Tu Codespace, la terminal, pwd, ls, cat. Tu primer archivo.
    ~15 min
  • 01
    Navegar el sistema de archivos cd, ls -la, find, mkdir, mv, rm. Encontrar un archivo oculto.
    ~25 min
  • 02
    Tu primer commit en Git init, status, add, commit, log, diff. Escribir un mensaje de commit decente.
    ~30 min
  • 03
    Tu primer script en Python python3, variables, print, input, funciones, correr un script desde la terminal.
    ~30 min
  • 04
    Ramas y fusiones en Git git branch, git switch, git merge, git log --graph. Tu primer merge de un ramo.
    ~30 min
  • 05
    Tu primera consulta SQL sqlite3, CREATE TABLE, INSERT, SELECT, WHERE. Biblioteca de libros con filtros.
    ~35 min
  • 06
    HTTP y APIs públicas curl, JSON, un script Python que llama a una API real y extrae campos.
    ~30 min

Total: 7 lecciones · ~3h 15min de tareas prácticas.
