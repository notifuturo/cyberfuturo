CAPÍTULO 01 · 14 min · CAPÍTULO PAGO · PRÉVIA

O terminal como linguagem.

Agora que você já sabe criar um arquivo, vamos aprender a se mover. Um sistema de arquivos é uma estrutura em forma de árvore — e os comandos para navegar essa árvore são os que você vai usar todos os dias da sua carreira.

A árvore

Um sistema de arquivos não é uma lista. É uma árvore. Tem uma raiz — em sistemas Unix, o / — e a partir dela tudo se ramifica em diretórios que contêm outros diretórios e, nas pontas, arquivos.

Quando você abre um terminal, você está em algum lugar dessa árvore. Esse “algum lugar” se chama diretório de trabalho. É relativo a ele que todos os comandos enxergam o mundo. Navegar significa escolher onde você quer estar.

▸
Regra mental
Caminhos que começam com / são absolutos — partem da raiz. Caminhos sem / inicial são relativos — partem de onde você está. Quase todo bug de “arquivo não encontrado” é um caminho relativo tratado como absoluto.

Os comandos que você vai usar todo dia

Você já conhece três do capítulo 00: pwd, ls e cat. Mais três, e você tem a caixa de ferramentas completa pra navegar qualquer projeto da sua carreira.

cd <caminho>
muda o diretório atual para <caminho>
cd ..
sobe um nível na árvore (.. significa “o pai”)
cd
volta pro seu diretório home — onde você começa quando o terminal abre
ls -la
lista tudo — inclusive arquivos ocultos (os que começam com ponto)
find . -name <padrão>
procura recursivamente a partir daqui por um nome que bata o padrão
user@cyberfuturo — zsh 
user@cyberfuturo:~$ cd curriculum/lessons/01-terminal
user@cyberfuturo:~/curriculum/lessons/01-terminal$ ls
lesson.md  lesson.es.md  test.py  workspace/
user@cyberfuturo:~/curriculum/lessons/01-terminal$ find workspace -name ".premio"
workspace/labirinto/passagem-a/dentro/.premio
$ █
