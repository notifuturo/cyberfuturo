A stack, camada por camada

A maioria dos cursos ensina Python da sintaxe para cima — e assume que você já sabe usar um terminal, um editor e o Git. A gente começa pela camada que você já reconhece e desce, uma ferramenta de cada vez, até o metal.

  1. CAMADA 6
    GUI — o que você já conhece

    VS Code, um navegador, arquivos. As mesmas interfaces que você usa todo dia.

  2. CAMADA 5
    Terminal & Bash

    O mesmo editor, agora com um terminal aberto. pwd, ls, cd, pipes, e o shell que você vai usar o resto da sua carreira.

  3. CAMADA 4
    Git & GitHub

    Cada exercício concluído é um commit de verdade. Você aprende Git usando, não estudando.

  4. CAMADA 3
    SQL com SQLite

    CREATE, INSERT, SELECT, joins — contra um banco de dados real que cabe em um único arquivo, direto dentro do seu Codespace.

  5. CAMADA 2
    Python do zero

    python3, variáveis, funções, arquivos, pip — com o interpretador já configurado no seu ambiente.

  6. CAMADA 1
    HTTP & APIs reais

    curl, respostas JSON, seu primeiro script chamando uma API pública. Agora você sabe como a Internet funciona por dentro.

Como funciona

user@cyberfuturo:~/curriculum$ ./cf start 1 ▸ Lição 01 — Primeiros passos no terminal Tarefa: crie um arquivo chamado ola.txt contendo "olá mundo". Dica: use echo com um redirecionamento > user@cyberfuturo:~/curriculum$ echo "olá mundo" > ola.txt user@cyberfuturo:~/curriculum$ ./cf check ✔ ola.txt existe ✔ conteúdo correto ✔ lição 01 concluída Próximo: ./cf next user@cyberfuturo:~/curriculum$ █
  1. Você abre o projeto no GitHub Codespaces

    Um clique no botão acima. O GitHub te dá uma máquina virtual com VS Code, um terminal, Python, Git e SQLite já configurados. Nada pra instalar.

  2. Você executa ./cf start para sua primeira lição

    Um programinha (escrito em Python puro, sem dependências) mostra a tarefa, as instruções e as dicas. As lições vivem no terminal porque o terminal é o que você veio aprender.

  3. Você faz a tarefa com comandos reais

    Cada exercício é uma pequena missão: criar um arquivo, achar um escondido, escrever um script Python, rodar uma consulta SQL. Você usa as ferramentas reais, sem simulação.

  4. Você executa ./cf check para validar

    O validador olha o estado real do seu sistema de arquivos, do seu repositório Git, ou da saída do seu programa. Ele te diz o que deu certo e o que está faltando.

  5. Cada lição concluída é um commit Git

    Você aprende Git sem perceber: cada conquista fica registrada no seu histórico. Quando terminar o curso, você vai ter um repositório real com 30+ commits reais, pronto pra colocar no seu currículo.

Currículo — versão 0.1

Um subconjunto pequeno, ensinado com profundidade. Quanto mais longo o currículo, menor a chance de você terminar. A gente escolhe lições que são indispensáveis, não "nice to have".

  • 00
    Boas-vindas & orientação Seu Codespace, o terminal, pwd, ls, cat. Seu primeiro arquivo.
    ~15 min
  • 01
    Navegar o sistema de arquivos cd, ls -la, find, mkdir, mv, rm. Achar um arquivo escondido.
    ~25 min
  • 02
    Seu primeiro commit em Git init, status, add, commit, log, diff. Escrever uma mensagem de commit decente.
    ~30 min
  • 03
    Seu primeiro script em Python python3, variáveis, print, input, funções, rodar um script pelo terminal.
    ~30 min
  • 04
    GitHub, branches e pull requests clone, push, branch, merge, resolver um conflito — em desenvolvimento.
    ~40 min
  • 05
    Sua primeira consulta SQL sqlite3, CREATE, INSERT, SELECT, WHERE, joins — em desenvolvimento.
    ~35 min
  • 06
    HTTP & APIs públicas com curl GET, POST, JSON, um script Python chamando uma API real — em desenvolvimento.
    ~30 min

Total atual: 4 lições · ~1h 40min. Em desenvolvimento: mais 3 lições.
