La pile, couche par couche

La plupart des cours enseignent Python depuis la syntaxe vers le haut — en supposant que tu sais déjà utiliser un terminal, un éditeur, et Git. Nous commençons par la couche que tu reconnais déjà et nous descendons, un outil à la fois, jusqu'au métal.

  1. COUCHE 6
    GUI — ce que tu connais déjà

    VS Code, un navigateur, des fichiers. Les mêmes interfaces que tu utilises tous les jours.

  2. COUCHE 5
    Terminal & Bash

    Le même éditeur, avec un terminal ouvert. pwd, ls, cd, pipes, et le shell que tu utiliseras toute ta carrière.

  3. COUCHE 4
    Git & GitHub

    Chaque exercice terminé est un vrai commit. Tu apprends Git en l'utilisant, pas en l'étudiant.

  4. COUCHE 3
    SQL avec SQLite

    CREATE, INSERT, SELECT, jointures — sur une vraie base de données qui tient dans un fichier, directement dans ton Codespace.

  5. COUCHE 2
    Python depuis zéro

    python3, variables, fonctions, fichiers, pip — avec l'interpréteur déjà configuré dans ton environnement.

  6. COUCHE 1
    HTTP & vraies APIs

    curl, réponses JSON, ton premier script qui appelle une API publique. Maintenant tu sais comment Internet fonctionne vraiment.

  1. Ouvre le projet dans GitHub Codespaces

    Un clic depuis le bouton ci-dessus. GitHub te donne une machine virtuelle avec VS Code, un terminal, Python, Git et SQLite déjà configurés. Rien à installer.

  2. Lance ./cf start pour ta première leçon

    Un petit programme (écrit en Python pur, sans dépendances) te montre la tâche, les instructions et les indices. Les leçons vivent dans le terminal parce que le terminal est ce que tu es venu apprendre.

  3. Fais la tâche avec de vraies commandes

    Chaque exercice est une petite mission : créer un fichier, en trouver un caché, écrire un script Python, exécuter une requête SQL. Tu utilises de vrais outils, pas des simulations.

  4. Lance ./cf check pour valider

    Le validateur regarde l'état réel de ton système de fichiers, de ton dépôt Git, ou de la sortie de ton programme. Il te dit ce qui a marché et ce qui manque.

  5. Chaque leçon terminée est un commit Git

    Tu apprends Git sans t'en rendre compte : chaque réussite est enregistrée dans ton historique. Quand tu termines le cours, tu as un vrai dépôt avec 30+ vrais commits, prêt à aller sur ton CV.

Programme — version 0.1

Un petit sous-ensemble, enseigné en profondeur. Plus le programme est long, moins tu as de chances de le terminer. On choisit des leçons indispensables, pas des "nice to have".

  • 00
    Bienvenue & orientation Ton Codespace, le terminal, pwd, ls, cat. Ton premier fichier.
    ~15 min
  • 01
    Naviguer le système de fichiers cd, ls -la, find, mkdir, mv, rm. Trouver un fichier caché.
    ~25 min
  • 02
    Ton premier commit Git init, status, add, commit, log, diff. Écrire un message de commit correct.
    ~30 min
  • 03
    Ton premier script Python python3, variables, print, input, fonctions, lancer un script depuis le terminal.
    ~30 min
  • 04
    Branches et fusions en Git git branch, git switch, git merge, git log --graph. Ta première fusion de branche.
    ~30 min
  • 05
    Ta première requête SQL sqlite3, CREATE TABLE, INSERT, SELECT, WHERE. Bibliothèque de livres avec filtres.
    ~35 min
  • 06
    HTTP et APIs publiques curl, JSON, un script Python qui appelle une vraie API et extrait des champs.
    ~30 min

Total : 7 leçons · ~3h 15min de tâches pratiques.
