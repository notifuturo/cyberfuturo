Methodology

Every CyberFuturo index has a versioned methodology document. Each document answers the same questions in the same order — so the indices can be compared on common criteria and audited against a consistent template.

The template

  1. What this measures — positive definition
  2. What this does not measure — explicit negative definition, usually longer than the positive
  3. Exact query — copy-pasteable, reproducible
  4. Pipeline — source file, language, dependencies
  5. Outputs — file paths and schemas
  6. Output schema — column types and invariants
  7. Known limitations — every known gap, explicitly stated
  8. Changelog — append-only version history
  9. How to reproduce — exact commands a reader can run
  10. Deprecation criteria — pre-committed conditions for retirement (BR-11)
  11. Contact — how to report an error

Indices

Live

  • arXiv AI research velocity arxiv-ai-velocity v0.1 · monthly cadence · 24-month window · status: live

    Monthly count of papers submitted to cs.AI or cs.LG on arXiv. The raw expansion rate of the AI/ML research surface area.

    Source: arXiv API · Raw CSV · Chart

In development

  • Global compute capex status: in development · methodology unpublished

    Combined quarterly capital expenditure on semiconductor fab and hyperscaler data center buildout, drawn from public SEC filings and vendor earnings disclosures.

  • Battery $/kWh status: in development · methodology unpublished

    Annual unsubsidized lithium-ion battery pack price tracked against historical trajectory, from public analyst reports with transparent provenance.

An index is never listed as "live" before its methodology is written, its pipeline is reproducible, and its first run has been committed. We do not announce dates for unreleased indices.

Enforcement

The methodology-presence rule is enforced mechanically. Our compliance script, scripts/lint_indices.py, scans the repository and fails the build if:

  • A canonical dataset exists under data/indices/ without a matching methodology document
  • A methodology document is missing any of the required template sections
  • A methodology document has no visible version tag
  • A methodology document lacks a deprecation-criteria section (violates BR-11)
  • A public methodology HTML page is missing for a published index

See the compliance report in the repository for the current state.
