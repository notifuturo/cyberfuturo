Editorial rules

Last updated: 2026-04-11 · Version: v0.1

These are the rules that govern every CyberFuturo brief, index, and sponsored slot. They exist because a measurement-first publication only compounds credibility when every piece of it is constrained by something other than the writer's preferences in the moment.

Every rule here is enforceable. Where we have an automated check, we say so.

Data & measurement

1. No forecasts

Every published claim is a measurement or a description of one. We never assert what a number is going to become, recommend trades, or claim causal relationships the data does not demonstrate.

Enforced by scripts/lint_editorial.py, which scans briefs and pages for forbidden vocabulary ("forecast", "predict", "will be", "on track to", etc.) and fails the build on any match.

2. Numbers come from public APIs, never from an LLM

Every number, chart value, and statistic in a CyberFuturo brief comes from a deterministic query against a public source, processed by a pipeline written in stdlib-only Python. Language models may draft or polish editorial prose. They may not write, infer, estimate, interpolate, or round any numerical value.

Enforced by scripts/lint_pipeline.py, which AST-parses the pipeline scripts and fails the build on any non-stdlib import or any known LLM client module.

3. Every number is traceable in three clicks

Brief → canonical CSV → methodology page with the exact query. If a reader cannot reach the underlying API query within three clicks of a headline number, we have violated this rule.

4. Methodology is versioned

Every index has a methodology document with a semver-like version. When a query, filter, or transformation changes, the version bumps and a changelog entry is recorded. Historical CSV values are preserved in git so older briefs remain verifiable against the exact numbers they cited at publication time.

5. Corrections appear in the next brief

When we find an error, we publish a correction in the next issue with a changelog entry. We never silently edit an archived brief.

Editorial

6. AI assistance is disclosed

Every brief that used LLM assistance for editorial prose says so, visibly, in the brief body — not only in metadata. The disclosure names the scope: whether AI touched prose, summaries, copy-edits, or nothing at all. Numbers are always produced by the deterministic pipeline, never by an LLM.

7. Structure is fixed

Every brief has exactly this structure:

  1. Header — issue number, date, cited indices, methodology link
  2. The number — pull-quote of the headline measurement
  3. Chart — the SVG with attribution
  4. What moved — specific observations, each with its own subheading
  5. What this is NOT — explicit delimitation of what the number does not measure
  6. How this was built — pointer to methodology and source code
  7. Next — 1–2 sentences on upcoming indices
  8. Sources — table of every claim with index slug and version
  9. Sponsor slot — optional, below the first editorial section, clearly labeled

8. A brief never contains

  • Predictions or investment advice
  • "Top 10" / "startups to watch" listicles
  • Political opinion
  • Personal speculation dressed as analysis
  • LLM-generated numbers, charts, tables, or metadata
  • Unattributed numbers
  • Engagement-bait CTAs or clickbait subject lines

Sponsors

9. One sponsor per brief, never programmatic

Every sponsored slot is a named company, directly sold, on a negotiated rate card. We do not run programmatic ads. We do not run display ads. We do not run affiliate banners above the editorial. At most one sponsor per issue.

10. Sponsors never appear above the editorial

The sponsor slot, if present, is located below the first editorial section — never above the fold, never in the header. The editorial above the fold never names or references the sponsor.

11. Editorial independence is absolute

A sponsor cannot review, influence, or request changes to editorial content beyond their own display copy. We do not let sponsors choose which indices we publish, which briefs reference their category, or which findings we emphasize. Sponsor agreements explicitly exclude editorial input.

Readers

12. Subscriber data is never shared

Your email address is used only to send you CyberFuturo briefs and to honor unsubscribe requests. We do not share it with sponsors. We do not aggregate it into segments that are sold. We do not pass it to marketing platforms beyond the one that sends the email. See the full privacy policy.

13. Unsubscribe is one click

Every email has a working unsubscribe link. Clicking it removes you from the list immediately with no "downgrade" flows, no retention pop-ups, no confirmation emails asking if you're sure.

Accountability

14. Git history is the audit trail

Every published brief, dataset, methodology version, and correction is preserved in git. We do not perform destructive history rewrites on published content except in explicit repo-visibility scenarios. Any reader in 2036 can clone the repo and verify the exact state at which a brief was published.

15. Every rule on this page is enforceable

The rules above are not aspirations. Each one has either an automated check (via scripts/compliance_check.py) or a documented manual-review gate. The compliance report lives in the repo at docs/compliance-report.md and is regenerated on every audit.

Rules change only through an explicit Architecture Decision Record. When they do, this page updates with a changelog entry below.
